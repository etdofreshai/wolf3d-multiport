ID_PM.C — Page Manager Engine

Purpose: Virtual memory system for the VSWAP file. Manages loading of wall textures, sprite frames, and digitized sounds on demand from disk into main memory, EMS, or XMS. Implements a page-based LRU cache with thrashing detection. The VSWAP file is divided into fixed-size 4096-byte pages.

Author: Jason Blochowiak, v1.0

Porting Notes

C SDL3
EMS/XMS code removed entirely. All pages loaded into malloc'd main memory. With modern RAM, the entire VSWAP file (700KB-1.5MB) can be loaded at startup. The paging/LRU logic is preserved for correctness but effectively always has enough cache.
C#
Read entire VSWAP into a byte[] or MemoryMappedFile. Page table becomes an array of ReadOnlyMemory<byte> slices.
TypeScript
Load VSWAP via fetch() into an ArrayBuffer. Slice into pages with DataView. All pages available immediately (no demand paging needed).
Love2D
Load VSWAP with love.filesystem.read(). Store as a Lua string or FFI buffer. Extract pages by offset/length on demand.
