WL_DEF.H — Main Definitions Game

Purpose: Master header for all Wolfenstein 3-D game code. Defines all game constants, data structures (player state, actor objects, door state, game state), enumerations (enemy types, weapon types, directions), macros, and extern declarations for cross-file globals. Every WL_*.C file includes this.

Key Data Structures

// Actor object - the main game entity
typedef struct objstruct {
    activetype  active;
    int         ticcount;
    classtype   obclass;
    statetype   *state;

    byte        flags;          // FL_SHOOTABLE, FL_BONUS, etc.
    long        distance;       // for AI: distance to player
    dirtype     dir;

    fixed       x, y;           // 16.16 fixed-point world position
    unsigned    tilex, tiley;   // tile position (x>>16, y>>16)
    byte        areanumber;

    int         viewx;          // screen x after projection
    unsigned    viewheight;     // projected height on screen
    fixed       transx, transy; // translated coordinates

    int         angle;          // facing angle
    int         hitpoints;
    long        speed;

    int         temp1, temp2, temp3;
    struct objstruct *next, *prev;
} objtype;
// Game state - persistent across save/load
typedef struct {
    int     difficulty;
    int     maession;
    int     episode, secretcount, treasurecount, killcount;
    int     secrettotal, treasuretotal, killtotal;
    long    TimeCount;
    long    killx, killy;
    int     bestweapon, weapon, chosenweapon;
    int     faceframe;
    int     attackframe, attackcount, weaponframe;
    int     keys;
    int     lives, health, ammo, score;
    int     nextextra;
    int     mapon;
    // ... weapon and item flags
} gametype;
// Door state
typedef struct {
    byte     tilex, tiley;
    boolean  vertical;
    byte     lock;
    doortype action;
    int      ticcount;
    unsigned position;       // 0=closed, 0xFFFF=open
} doorobj_t;

Porting Notes

C SDL3
Most definitions carry over directly. Remove far/_seg qualifiers. The fixed type stays as long (32-bit). SCREENSEG constant no longer meaningful but kept for compatibility. MK_FP and segment math eliminated.
C#
Structs become C# struct or class. The objtype linked list can use LinkedList<T> or manual next/prev references. Enums map directly. Fixed-point uses int (32-bit).
TypeScript
All structs become TypeScript interfaces or classes. Fixed-point arithmetic uses regular number (float64) with appropriate scaling, or a dedicated fixed-point helper class. Enums with const enum.
Love2D
Structs become Lua tables. Constants as module-level variables. The linked list for actors becomes a Lua array with index-based iteration. Fixed-point may use integer math via math.floor or floating point.
