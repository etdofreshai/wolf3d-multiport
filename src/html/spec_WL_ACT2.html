WL_ACT2.C — Actors Part 2: Enemies & AI Game

Purpose: The largest source file. Contains all enemy spawning, AI think functions, state tables, boss behaviors, projectile handling, death animations, and the BJ victory sequence. Each enemy type has spawn, stand, path, chase, shoot, and die behaviors defined as state machine entries.

Enemy Spawning

  • void SpawnStand(enemy_t which, int tilex, int tiley, int dir) — Spawn stationary guard
  • void SpawnPatrol(enemy_t which, int tilex, int tiley, int dir) — Spawn patrolling guard
  • void SpawnDeadGuard(int tilex, int tiley) — Spawn dead decoration guard
  • void SpawnBoss(int tilex, int tiley) — Spawn Hans Grosse
  • void SpawnGretel(int tilex, int tiley) — Spawn Gretel Grosse
  • void SpawnSchabbs(int tilex, int tiley) — Spawn Dr. Schabbs
  • void SpawnFakeHitler(int tilex, int tiley) — Spawn Fake Hitler (mecha)
  • void SpawnHitler(int tilex, int tiley) — Spawn real Hitler
  • void SpawnGift(int tilex, int tiley) — Spawn Otto Giftmacher
  • void SpawnFat(int tilex, int tiley) — Spawn General Fettgesicht
  • void SpawnSpectre(int tilex, int tiley) — Spawn Spectre ghost
  • void SpawnGhosts(int which, int tilex, int tiley) — Spawn ghost enemies

AI Think Functions

  • void T_Stand(objtype *ob) — Standing guard: check for player sighting
  • void T_Path(objtype *ob) — Patrolling: follow path markers
  • void T_Chase(objtype *ob) — Chasing player: pathfinding toward player
  • void T_DogChase(objtype *ob) — Dog chase: similar to T_Chase but dogs bite
  • void T_Shoot(objtype *ob) — Shooting at player
  • void T_Bite(objtype *ob) — Dog bite attack
  • void T_Projectile(objtype *ob) — Projectile movement (needles, rockets, fireballs)
  • void T_Schabb(objtype *ob) — Dr. Schabbs boss AI
  • void T_Gift(objtype *ob) — Otto Giftmacher boss AI
  • void T_Fat(objtype *ob) — Fettgesicht boss AI
  • void T_Fake(objtype *ob) — Fake Hitler / Mecha-Hitler AI
  • void T_Ghosts(objtype *ob) — Ghost movement (pass through walls)

Action Functions

  • void A_DeathScream(objtype *ob) — Play death sound based on enemy type
  • void A_Smoke(objtype *ob) — Spawn smoke puff effect
  • void A_HitlerMorph(objtype *ob) — Mecha-Hitler to real Hitler transformation
  • void A_StartDeathCam(objtype *ob) — Trigger death camera for boss kills
  • void SpawnBJVictory(void) — Spawn BJ Blazkowicz victory animation
  • boolean ProjectileTryMove(objtype *ob) — Check if projectile can move to new position
  • boolean CheckPosition(objtype *ob) — Verify actor position is valid

Dependencies

  • WL_DEF.H — Game definitions, state tables
  • WL_STATE.C — AI movement helpers (SelectChaseDir, MoveObj, etc.)
  • WL_AGENT.C — Player damage (TakeDamage)
  • ID_SD.C — Sound effects

Porting Notes

C SDL3
Pure game logic. The state tables (arrays of statetype structs with think/action function pointers) carry over directly. No platform-specific code.
TypeScript
State tables can use objects with function references. The large number of enemy types and states makes this a tedious but mechanical translation. Function pointers in statetype become method references.
