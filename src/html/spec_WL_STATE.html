WL_STATE.C — Actor State Machine Game

Purpose: Core actor AI infrastructure. Provides the state machine engine (state transitions, think function dispatch), movement and collision for AI actors, pathfinding direction selection (chase, dodge, run), line-of-sight checking, enemy alerting/sighting, and the kill/damage system.

Functions — State Machine

  • void SpawnNewObj(unsigned tilex, unsigned tiley, statetype *state) — Allocate actor and set initial state
  • void NewState(objtype *ob, statetype *state) — Transition actor to a new state

Functions — Movement

  • void MoveObj(objtype *ob, long move) — Move actor in its current direction by given distance
  • boolean TryWalk(objtype *ob) — Check if actor can walk one tile in its direction
  • void SelectChaseDir(objtype *ob) — Choose direction toward player (A* lite)
  • void SelectDodgeDir(objtype *ob) — Choose direction to dodge (perpendicular to player)
  • void SelectRunDir(objtype *ob) — Choose direction away from player (retreat)
  • void SelectPathDir(objtype *ob) — Follow patrol path markers

Functions — Combat

  • void KillActor(objtype *ob) — Kill an actor: set death state, drop item, update kill count
  • void DamageActor(objtype *ob, unsigned damage) — Apply damage to actor, kill if hitpoints reach 0
  • void DropItem(stat_t itemtype, int tilex, int tiley) — Drop a pickup item when actor dies

Functions — Sighting

  • boolean CheckLine(objtype *ob) — Line-of-sight check from actor to player (traces through doors/tiles)
  • boolean CheckSight(objtype *ob) — Full sight check (area connectivity + line of sight)
  • void FirstSighting(objtype *ob) — Actor sees player for first time: alert, play sound, switch to chase
  • boolean SightPlayer(objtype *ob) — Attempt to sight the player (random chance + CheckSight)

Dependencies

  • WL_DEF.H — Game definitions (objtype, statetype, dirtype)
  • WL_PLAY.C — tilemap, actorat, actor list management
  • WL_ACT1.C — ConnectAreas, PlaceItemType (for dropped items)
  • ID_SD.C — Alert/death sounds

Porting Notes

C SDL3
Pure game logic with fixed-point math. All functions are portable. The state machine pattern (statetype with function pointers for think/action) is a clean, portable design.
TypeScript
The state machine translates well. statetype becomes an object with think and action as function references. CheckLine ray tracing through the tile grid uses integer stepping — straightforward to translate.
