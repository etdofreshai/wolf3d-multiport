WL_SCALE.C — Sprite/Column Scaling Game

Purpose: Generates compiled (self-modifying) scaling routines at runtime for drawing wall texture columns and sprites at various heights. The original creates x86 machine code in memory that draws scaled columns as fast as possible. Also contains the sprite drawing entry points.

Functions

  • void SetupScaling(int maxscaleheight) — Generate compiled scaling routines for all needed heights
  • unsigned BuildCompScale(int height, memptr *finalspot) — Build one compiled scaler for a specific height
  • void ScaleShape(int xcenter, int shapenum, unsigned height) — Draw a scaled sprite centered at xcenter
  • void SimpleScaleShape(int xcenter, int shapenum, unsigned height) — Simplified sprite scaler (for weapon overlay)

Dependencies

  • WL_DEF.H — Game definitions
  • ID_PM.C — PM_GetPage for texture data
  • ID_MM.C — MM_GetPtr for scaler allocation
  • ID_VL.C — Screen buffer access
Heavy porting effort: The compiled scalers generate x86 machine code at runtime. This is the single most DOS-specific technique in the codebase. Every port must replace this with a general-purpose C/language scaling loop.

Porting Notes

C SDL3
Replace compiled scalers with C scaling loops. For each column: calculate source texel step based on height, iterate down the destination column, sample the source texture at the appropriate texel. Wall columns are 64 pixels tall in the texture. Sprites use the shape format (column-based with post runs). Performance is adequate on modern CPUs without self-modifying code.
C#
Same approach: C# scaling loops with Span<byte> for the inner loop. The texel stepping can use fixed-point or floating-point.
TypeScript
Scaling loops writing to Uint8Array or Uint32Array (pre-paletted). This is the inner-most rendering loop and benefits from keeping the typed array operations tight.
Love2D
Per-pixel scaling via ImageData, or use love.graphics.draw with quad sub-rectangles for each column slice. The latter leverages GPU acceleration and may be faster than CPU pixel loops.
