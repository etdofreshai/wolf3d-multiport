WL_TEXT.C — Text Display Game

Purpose: Rich text layout and display engine for in-game text screens (help, story text between episodes). Parses a simple markup language with commands for pictures, timed displays, centering, and page breaks. Used for the "Read This" help screens and end-of-episode story pages.

Functions

  • void ShowArticle(char far *article) — Display a multi-page text article with navigation
  • void HelpScreens(void) — Show the help/instructions text
  • void EndText(void) — Show end-of-episode story text
  • void PageLayout(boolean shownumber) — Lay out one page of text with word wrapping
  • void BackPage(void) — Navigate to previous page
  • void HandleCommand(void) — Process a text markup command (^P, ^C, ^T, ^E, ^G, ^L)
  • void HandleWord(void) — Process one word, handling line/page overflow
  • void HandleCtrls(void) — Process control characters (newline, tab)
  • void NewLine(void) — Move to next line, checking for page overflow
  • void RipToEOL(void) — Skip to end of current line in text data
  • int ParseNumber(void) — Parse a numeric value from text data
  • void ParsePicCommand(void) — Parse ^P picture display command
  • void ParseTimedCommand(void) — Parse ^T timed display command
  • void TimedPicCommand(void) — Execute timed picture display
  • void CacheLayoutGraphics(void) — Pre-cache graphics referenced by text

Text Markup Commands

CommandDescription
^PDisplay a picture at current position
^CCenter text on this line
^TTimed display (show picture for N tics)
^EEnd of article
^GChange font color
^LPage break (new page)
^BSet background color

Dependencies

  • WL_DEF.H — Game definitions
  • ID_VH.C — Text and picture drawing
  • ID_CA.C — Graphics caching
  • ID_IN.C — Navigation input (PageUp/PageDown/Esc)

Porting Notes

C SDL3
Direct translation. The text parser operates on a character pointer through the article data. Page navigation is a simple blocking loop. The ^T timed command uses VBL waits.
TypeScript
The article viewer becomes an interactive component with page state. The text parser can be rewritten as a tokenizer that processes commands into a display list.
