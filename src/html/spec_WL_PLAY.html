WL_PLAY.C — Play Loop Game

Purpose: The inner gameplay loop. Runs each frame: polls controls, processes all actors, handles music, manages palette shifts (damage/bonus flashes), checks debug keys, and calls ThreeDRefresh for rendering. Also contains actor list management and the control polling subsystem.

Porting Notes

C SDL3
PlayLoop works as-is since it already loops on a frame-by-frame basis. Control polling adapts to SDL events. Palette shifts apply to the sdl_palette array. The actor list management is pure C data structure manipulation.
TypeScript
PlayLoop becomes a per-frame function called from requestAnimationFrame. The while(playstate == ex_stillplaying) loop becomes implicit in the frame callback. All control polling uses DOM events cached from the event listener.
