WL_AGENT.C — Player Agent Game

Purpose: All player-specific logic: movement and collision, weapon management, combat (knife/gun attacks, hit detection), status bar HUD drawing (health, ammo, score, keys, face), item pickup, and the player think function (T_Player) that runs each tic.

Functions — Player Think

  • void T_Player(objtype *ob) — Main player think: movement, weapon check, use/fire
  • void T_Attack(objtype *ob) — Player attack think: animate weapon, fire projectile
  • void ControlMovement(objtype *ob) — Convert control input to movement and rotation
  • void CheckWeaponChange(void) — Check if player wants to switch weapons (1-4 keys)

Functions — Movement & Collision

  • boolean TryMove(objtype *ob) — Attempt to move player, checking tile/actor collisions
  • void ClipMove(objtype *ob, long xmove, long ymove) — Move with wall sliding (try X, try Y independently)
  • void Thrust(int angle, long speed) — Apply thrust in a direction, with ClipMove
  • void VictoryTile(void) — Handle stepping on exit tile

Functions — Combat

  • void Cmd_Fire(void) — Begin weapon fire sequence
  • void Cmd_Use(void) — Use action (open doors, push walls)
  • void KnifeAttack(objtype *ob) — Knife melee attack: check range, deal damage
  • void GunAttack(objtype *ob) — Gun hitscan attack: trace ray, check hit, apply damage
  • void VictorySpin(void) — Victory spin animation when completing an episode

Functions — HUD / Status Bar

  • void DrawFace(void) — Draw BJ's face in the status bar
  • void UpdateFace(void) — Update face expression based on health/damage
  • void DrawHealth(void) — Draw health number
  • void DrawAmmo(void) — Draw ammo count
  • void DrawScore(void) — Draw score
  • void DrawLevel(void) — Draw level number
  • void DrawLives(void) — Draw lives count
  • void DrawKeys(void) — Draw key icons
  • void DrawWeapon(void) — Draw weapon icon in status bar
  • void StatusDrawPic(unsigned x, unsigned y, unsigned picnum) — Draw a picture in the status bar area
  • void LatchNumber(int x, int y, int width, long number) — Draw a number using latch tile digits

Functions — Items & Pickups

  • void GetBonus(statobj_t *check) — Pick up a bonus item (ammo, health, treasure, keys, weapons)
  • void GiveWeapon(int weapon) — Give player a weapon
  • void GiveAmmo(int ammo) — Add ammo
  • void GiveKey(int key) — Give a key
  • void GiveExtraMan(void) — Award extra life
  • void GivePoints(long points) — Add score points (may trigger extra life)
  • void TakeDamage(int points, objtype *attacker) — Player takes damage
  • void HealSelf(int points) — Heal player
  • void SpawnPlayer(int tilex, int tiley, int dir) — Initialize player object at map start

Dependencies

  • WL_DEF.H — Game definitions
  • WL_STATE.C — CheckLine (line-of-sight for gun attacks)
  • WL_ACT1.C — OperateDoor, PushWall (for Use action)
  • ID_VH.C — LatchDrawPic for status bar
  • ID_SD.C — Weapon/pickup sounds

Porting Notes

C SDL3
Pure game logic. Movement uses fixed-point math (portable). Status bar drawing calls through the VH/VL layer. GunAttack hitscan traces through the tile grid — straightforward math.
TypeScript
Fixed-point collision detection needs care to avoid floating-point precision issues. Consider using integer math (|0 truncation) for TryMove/ClipMove to match original behavior exactly.
