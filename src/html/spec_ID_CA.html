ID_CA.C — Cache Manager Engine

Purpose: Manages loading and caching of all game assets from the three archive files: VGAGRAPH (graphics), MAPHEAD/GAMEMAPS (level maps), and AUDIOT/AUDIOHED (audio). Implements Huffman decompression, Carmack compression, and RLEW expansion for the packed asset data. Provides a level-based caching system where assets are marked as needed and batch-loaded.

Author: John Carmack (customized for Wolf3D)

Porting Notes

C SDL3
File I/O switches from DOS open/read to standard fopen/fread. The far/huge pointer qualifiers are removed. Decompression algorithms (Huffman, Carmack, RLEW) are portable C and need minimal changes. The segment-based memory pointers become regular pointers.
C#
FileStream/BinaryReader for file I/O. Decompression algorithms translated to use byte[] and Span<byte>. Graphics chunks stored as managed arrays.
TypeScript
Asset files loaded via fetch() into ArrayBuffer. Decompression uses DataView for word-level access. All decompression must handle endianness (original is little-endian x86).
Love2D
love.filesystem for file access. Decompression in Lua using string.byte/char or FFI buffers. Performance-sensitive decompression may benefit from LuaJIT FFI.
