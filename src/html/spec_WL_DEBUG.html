WL_DEBUG.C — Debug Tools Game

Purpose: Development and debugging tools accessed via the backspace+key combination during gameplay. Includes god mode, noclip, warp to level, memory display, object counting, overhead map view, shape/sprite viewer, and various stat displays.

Functions

  • int DebugKeys(void) — Main debug key dispatcher (Tab+key combinations)
  • void DebugMemory(void) — Display memory usage statistics
  • void CountObjects(void) — Count and display active/static objects
  • void PicturePause(void) — Freeze frame with picture display
  • void ShapeTest(void) — Interactive sprite/shape viewer
  • void ViewMap(void) — Overhead map viewer with pan/zoom
  • void OverheadRefresh(void) — Render the overhead map view

Dependencies

  • WL_DEF.H — Game definitions
  • WL_PLAY.C — godmode, noclip flags; tilemap, actorat arrays

  • ID_IN.C — Input for debug commands

Porting Notes

C SDL3
Debug functions are mostly display code. The overhead map renderer draws colored blocks to represent tiles — simple to port. Memory debug display adapts to show malloc usage instead of segment counts. Shape viewer loads pages from PM and draws them.
TypeScript
Debug tools can be enhanced with browser developer tools integration. The overhead map is a natural fit for a canvas overlay. Memory display shows JS heap stats.
