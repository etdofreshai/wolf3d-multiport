ID_VH.C — Video High-Level Engine

Purpose: High-level video routines built on top of ID_VL. Provides proportional font rendering, picture drawing, double-buffer update management, the FizzleFade transition effect, and Wolf3D-specific routines like LatchDrawPic and LoadLatchMem. This is the layer that game code calls for 2D drawing.

Porting Notes

C SDL3
VL_MungePic is simplified since there are no VGA planes. The VWB_* double-buffer wrappers still mark dirty regions but the update mechanism can be simplified to just redraw the whole 320x200 buffer each frame. FizzleFade works on the linear buffer using the same LFSR algorithm but writes individual pixels instead of VGA plane-masked bytes. LoadLatchMem stores deplanarized graphics into the latchmem[] array.
C#
Font rendering can use the same pixel-by-pixel approach or be accelerated with span copies. The update block tracking can be simplified to full-screen redraw. FizzleFade LFSR algorithm is portable as-is.
TypeScript
All drawing writes to a Uint8Array framebuffer. Font data extracted from graphics lumps at load time. FizzleFade must be converted from a blocking loop to an async/frame-callback pattern.
Love2D
Drawing primitives write to ImageData. FizzleFade needs coroutine-based yielding per frame instead of a blocking loop. Font rendering pixel-by-pixel via imageData:setPixel().
