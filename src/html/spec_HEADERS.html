Asset & Configuration Headers Headers

Purpose: These header files define enumeration constants for graphics, sound, and map assets, as well as version-specific configuration. They are generated by id Software's asset pipeline tools and define the symbolic names that map to numeric chunk indices in the data files.

Audio Headers

AUDIOWL6.H / AUDIOWL1.H / AUDIOSOD.H / AUDIOSDM.H

Define enumerated sound effect and music track names for each game version. Each enum value maps to a chunk index in the AUDIOT/AUDIOHED files.

// Example from AUDIOWL6.H
typedef enum {
    HITWALLSND,          // 0
    SELECTWPNSND,        // 1
    SELECTITEMSND,       // 2
    HEARTBEATSND,        // 3
    // ... hundreds of sound entries
    LASTSOUND
} soundnames;

Each game version (WL1=shareware, WL6=registered, SOD=Spear of Destiny, SDM=Spear demo) has its own audio header with version-appropriate sounds.

BUDIOWL6.H / BUDIOSOD.H

Alternate audio headers (possibly for different audio configurations or builds).

Graphics Headers

GFXV_WL6.H / GFXV_WL1.H / GFXV_SOD.H / GFXV_SDM.H

Define enumerated graphic chunk names. The VGAGRAPH file contains fonts, pictures (menu art, status bar elements), and tile sets. Each entry maps to a chunk index.

// Example structure
typedef enum {
    // Fonts
    STARTFONT,
    STARTFONTM = STARTFONT + 2,
    // Pics
    STARTPICS,
    H_BJPIC = STARTPICS,
    H_CASTLEPIC,
    H_KEYBOARDPIC,
    // ... many picture entries
    // Tiles
    STARTTILE8,
    STARTTILE16,
    NUMCHUNKS
} graphicnums;

GFXE_WL6.H / GFXE_WL1.H / GFXE_SOD.H

EGA graphics versions (not used in VGA builds but present in source).

Map Headers

MAPSWL6.H / MAPSWL1.H / MAPSSOD.H / MAPSSDM.H / MAPSWLF.H

Define map count and map names for each version.

// Example
#define NUMMAPS     60      // WL6 has 60 maps (6 episodes x 10 levels)

// Map plane structure:
// Plane 0: Wall/floor tile values (what to draw)
// Plane 1: Object/info values (where to spawn things)
// Plane 2: Additional info (floor codes, etc.)

Version Headers

VERSION.H / WOLFVER.H / WOLF1VER.H / SODVER.H / SDMVER.H / etc.

Define compile-time switches that select between game versions:

VersionDefineDescription
WL1UPLOADWolfenstein 3-D shareware (episode 1 only)
WL6GOODTIMESWolfenstein 3-D registered (all 6 episodes)
SODSPEARSpear of Destiny
SDMSPEARDEMOSpear of Destiny demo

These control which audio, graphics, and map headers are included, and enable/disable version-specific features (extra bosses, different levels, etc.).

SPANVER.H / SPANISH.H

Spanish localization defines and translated string tables.

Engine Interface Headers

ID_HEAD.H / ID_HEADS.H

ID_HEAD.H defines basic types (boolean, byte, word, longword, fixed, memptr) and VGA register constants. ID_HEADS.H is the master include that pulls in all engine headers in the correct order.

ID_VL.H / ID_VH.H / ID_IN.H / ID_SD.H / ID_MM.H / ID_PM.H / ID_CA.H / ID_US.H

Public API declarations for each engine subsystem. Define types, function prototypes, and constants used across modules.

WL_MENU.H

Menu system types: CP_iteminfo, CP_itemtype, CustomCtrls, and menu item arrays.

FOREIGN.H / F_SPEAR.H

FOREIGN.H: English string table. F_SPEAR.H: Spear of Destiny overrides (different level names, boss names, etc.).

Assembly Files

*.ASM Files

The following assembly files provide performance-critical routines:

FilePurpose
C0.ASMC runtime startup code (before main)
ID_VL_A.ASMVGA: VideoID, SetCRTC, SetScreen, WaitVBL
ID_VH_A.ASMVH_UpdateScreen (page flip)
ID_SD_A.ASMSound: SetDS, timer ISR wrappers
ID_US_A.ASMRandom number generator
WL_ASM.ASMGame-specific assembly helpers
WL_DR_A.ASMRaycaster inner loop (DDA stepping)
JABHACK.ASMJason Blochowiak's interrupt hack
H_LDIV.ASM32-bit division for 16-bit compiler
WHACK_A.ASMFast memory copy/set routines

In the SDL3 port, these are replaced with equivalent C functions (e.g., WL_ASM.C, WL_DR_A.C, ID_VL_A.C, ID_VH_A.C, ID_US_A.C).

Porting Notes

C SDL3
All headers carry over with minimal changes. Remove far/_seg/huge qualifiers from type definitions. The version-selection mechanism works the same way via #define. Assembly files are replaced with C equivalents already present in the SDL3 port.
C#
Enums translate directly. Type definitions become C# types (boolean = bool, byte = byte, word = ushort, longword = uint, fixed = int). Version selection can use preprocessor symbols or runtime configuration.
TypeScript
Enums become const enum or frozen objects. Type aliases or interfaces replace C typedefs. Version selection at build time (separate builds) or runtime (config object).
Love2D
All constants become Lua module-level variables or tables. No preprocessor — version selection via runtime configuration table.
