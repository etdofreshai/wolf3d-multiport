WL_MENU.C — Menu System Game

Purpose: The complete menu system: main menu, new game (episode/difficulty selection), sound options, control configuration (keyboard/mouse/joystick binding), view size adjustment, load/save game, and quit confirmation. Uses a data-driven menu item system with the HandleMenu generic handler.

Functions — Menu Pages

  • void US_ControlPanel(byte scancode) — Main entry point to the control panel (menu system)
  • void DrawMainMenu(void) — Draw the main menu screen
  • void CP_NewGame(void) — New game menu (episode + difficulty)
  • void CP_Sound(void) — Sound options menu
  • void CP_Control(void) — Control options menu
  • int CP_LoadGame(int quick) — Load game menu
  • int CP_SaveGame(int quick) — Save game menu
  • void CP_ChangeView(void) — View size adjustment
  • void CP_Quit(void) — Quit confirmation
  • void CP_ViewScores(void) — High score display
  • void CP_ReadThis(void) — Read This! help pages
  • int CP_EndGame(void) — End current game confirmation
  • int CP_CheckQuick(unsigned scancode) — Quick key handlers (F-keys for quick save/load/sound/quit)

Functions — Menu Engine

  • void DrawMenu(CP_iteminfo *item_i, CP_itemtype far *items) — Render menu items

Porting Notes

C SDL3
The menu system is self-contained UI code. The main challenge is that HandleMenu uses a blocking loop — works fine in SDL since we can pump events. Control configuration needs to map SDL key names instead of DOS scan code names. Joystick calibration uses SDL gamepad API.
TypeScript
The blocking menu loops must become async state machines. Each menu page becomes a state with enter/update/draw phases. The animated cursor gun and menu transitions need frame-by-frame handling.
