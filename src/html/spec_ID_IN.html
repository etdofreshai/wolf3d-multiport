ID_IN.C — Input Manager Engine

Purpose: Handles all input devices: keyboard (via INT 9 ISR), mouse (via INT 33h driver), and joystick (via port 0x201 polling). Provides a unified control abstraction (ControlInfo) and demo recording/playback through serialized control data.

Author: Jason Blochowiak, v1.0d1

Side-by-Side: Keyboard Handling

C Original (DOS)
// Hardware interrupt service routine
static void interrupt INL_KeyService(void)
{
    byte k = inportb(0x60);  // Read scan code
    // Acknowledge to keyboard controller
    outportb(0x61,
        (temp = inportb(0x61)) | 0x80);
    outportb(0x61, temp);

    if (k == 0xe0)
        special = true;
    else if (k & 0x80)     // Key release
        Keyboard[k & 0x7f] = false;
    else                     // Key press
    {
        LastScan = k;
        Keyboard[k] = true;
        // ASCII lookup...
    }
    outportb(0x20, 0x20);  // EOI to PIC
}
C SDL3 Port
// Called from IN_PumpEvents() each frame
// SDL translates OS key events to SDL scancodes
void IN_HandleKeyEvent(SDL_Event *ev)
{
    byte sc = SDL_ScancodeToScancode(
        ev->key.scancode);

    if (ev->type == SDL_EVENT_KEY_DOWN)
    {
        LastScan = sc;
        Keyboard[sc] = true;
        // ASCII from SDL key sym
        LastASCII = (char)ev->key.key;
    }
    else // SDL_EVENT_KEY_UP
    {
        Keyboard[sc] = false;
    }
}
// Note: SDL_ScancodeToScancode maps SDL scancodes
// to the DOS scan code values the game expects.

Porting Notes

C SDL3
Replace INT 9 keyboard ISR with SDL event polling. Map SDL scancodes to DOS scan codes so the Keyboard[] array stays compatible. Mouse via SDL_GetRelativeMouseState. Joystick via SDL gamepad API. The demo record/playback system is unchanged since it operates on the abstract ControlInfo.
C#
Same approach as C SDL3 using SDL3-CS bindings. The Keyboard[] array becomes a bool[]. Consider using an enum for scan codes instead of raw byte values.
TypeScript
Use keydown/keyup DOM events. Map KeyboardEvent.code strings to DOS scan code numbers. Mouse via pointer lock API. Joystick via Gamepad API. All blocking waits (IN_WaitForKey, IN_Ack) must become async.
Love2D
Use love.keypressed/love.keyreleased callbacks. Map Love2D key constants to DOS scan codes. Mouse via love.mouse. Joystick via love.joystick. Blocking waits become coroutine yields.
