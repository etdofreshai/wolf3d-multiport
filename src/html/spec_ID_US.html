ID_US_1.C — User Manager Engine

Purpose: General user-interface support routines. Handles windowed text printing, parameter checking, hard error handling, and maintains the high score table and save game slots. Provides the text printing system used by menus and in-game messages.

Author: Jason Blochowiak, v1.1d1

Porting Notes

C SDL3
Mostly portable C. The DOS hard error handler (USL_HardError) is removed. US_CheckParm works with standard argc/argv. The text printing system calls through function pointers (USL_MeasureString, USL_DrawString) that work unchanged.
C#
Direct translation. String handling uses string or ReadOnlySpan<char>. The function-pointer callbacks become delegates or interface methods.
TypeScript
String handling natural in JS. Window drawing and text output write to the framebuffer. Command-line parameter checking adapts to URL parameters or startup config.
Love2D
Text printing and window drawing translate directly. Lua string handling simplifies much of the code. Save games use love.filesystem for persistence.
