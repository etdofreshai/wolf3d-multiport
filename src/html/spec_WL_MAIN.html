WL_MAIN.C — Main Entry Point Game

Purpose: Contains main(), initialization, shutdown, the demo loop (title screen / credits / demo playback cycle), configuration read/write, save/load game serialization, projection table setup, and the Quit() error handler. This is the top-level file that ties everything together.

Author: John Carmack

Functions

  • void main(void) — Entry point: parse args, InitGame, DemoLoop
  • void InitGame(void) — Initialize all subsystems in order (MM, PM, CA, VL, VH, IN, SD, US)
  • void ShutdownId(void) — Shut down all subsystems in reverse order
  • void DemoLoop(void) — Main attract loop: signon, title, credits, auto-demo, repeat
  • void Quit(char *error) — Clean shutdown with optional error message
  • void ReadConfig(void) — Read CONFIG.WL6: controls, sound settings, view size
  • void WriteConfig(void) — Write CONFIG.WL6
  • boolean SaveTheGame(int file, int x, int y) — Serialize full game state to file
  • boolean LoadTheGame(int file, int x, int y) — Deserialize game state from file
  • void NewGame(int difficulty, int episode) — Reset game state for new game
  • void BuildTables(void) — Precompute sin/cos/tangent tables
  • void CalcProjection(long focal) — Set up projection constants for current view size
  • void SetupWalls(void) — Compute wall tile mapping
  • boolean SetViewSize(unsigned width, unsigned height) — Change viewport dimensions
  • void NewViewSize(int width) — Apply new view size setting
  • void ShowViewSize(int width) — Preview view size change
  • void SignonScreen(void) — Display the signon (loading) screen
  • void FinishSignon(void) — Complete signon display
  • long DoChecksum(byte far *source, unsigned size, long checksum) — Compute checksum for save game verification
  • void InitDigiMap(void) — Set up digitized sound mapping table
  • boolean MS_CheckParm(char far *check) — Check for command-line parameter
  • void Patch386(void) — Patch code for 386 optimized routines

Dependencies

  • WL_DEF.H — All game definitions
  • All ID_* modules — Initialized/shutdown from here
  • All WL_* modules — Called from DemoLoop and GameLoop

Porting Notes

C SDL3
main() becomes SDL_main(). DemoLoop runs inside a standard event-driven loop. Command-line parsing uses standard argc/argv. Save/load uses fopen/fwrite/fread. The Patch386 function becomes a no-op. BuildTables is identical (fixed-point trig).
TypeScript
The blocking DemoLoop must be converted to an async state machine or requestAnimationFrame-based loop. Save/load uses localStorage or IndexedDB. Sin/cos tables can use Math.sin or precomputed arrays.
