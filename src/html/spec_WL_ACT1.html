WL_ACT1.C — Actors Part 1: Doors & Statics Game

Purpose: Handles static objects (decorations, pickups), door mechanics (spawning, opening, closing, state machine), area connectivity for sound propagation, and push-wall movement. Contains the non-enemy actor systems.

Functions

  • void InitStaticList(void) — Reset static object array
  • void SpawnStatic(int tilex, int tiley, int type) — Place a static object (lamp, barrel, food, ammo, etc.)
  • void PlaceItemType(int itemtype, int tilex, int tiley) — Place a pickup item (called when enemies drop items)
  • void InitDoorList(void) — Reset door array
  • void SpawnDoor(int tilex, int tiley, boolean vertical, int lock) — Create a door at map position
  • void OpenDoor(int door) — Begin opening a door
  • void CloseDoor(int door) — Begin closing a door
  • void OperateDoor(int door) — Toggle door (checks keys, starts open/close)
  • void DoorOpen(int door) — Door fully open state handler
  • void DoorOpening(int door) — Door sliding open animation
  • void DoorClosing(int door) — Door sliding closed animation
  • void MoveDoors(void) — Update all active doors (called each tic)
  • void PushWall(int checkx, int checky, int dir) — Start pushing a secret wall
  • void MovePWalls(void) — Update push-wall movement (called each tic)
  • void InitAreas(void) — Reset area connectivity
  • void ConnectAreas(void) — Rebuild area connectivity graph (for sound propagation through open doors)
  • void RecursiveConnect(int areanumber) — Flood-fill area connectivity

Dependencies

  • WL_DEF.H — Game definitions (doorobj_t, statobj_t)
  • WL_AGENT.C — Player key checking for locked doors
  • ID_SD.C — Door sound effects

Porting Notes

C SDL3
Pure game logic — translates directly with no platform-specific changes. Door position uses a 0-0xFFFF range for smooth animation. Push-wall movement is fixed-point tile stepping.
TypeScript
Direct translation. Door and push-wall state machines are simple switch/case patterns. The area connectivity flood-fill is a standard recursive algorithm.
