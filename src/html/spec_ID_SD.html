ID_SD.C — Sound Manager Engine

Purpose: Manages all sound hardware: PC speaker, AdLib (OPL2) FM synthesis for music and sound effects, Sound Blaster for digitized audio, and the Disney Sound Source. Drives the timer interrupt (INT 8) which also provides the global TimeCount tick counter used throughout the game for timing.

Author: Jason Blochowiak, v1.2

Critical: The timer ISR is responsible for incrementing TimeCount, which is the master clock for the entire game. All timing — frame rate, animation, sound sequencing, demo playback — depends on this 70Hz counter. The SDL3 port replaces this with SDL_GetTicks() scaled appropriately.

Porting Notes

C SDL3
Replace AdLib register programming with OPL3 emulation (or pre-rendered audio). Sound Blaster DMA replaced with SDL audio callbacks. TimeCount derived from SDL_GetTicks() at 70Hz granularity. PC speaker sounds can be synthesized as square waves. IMF music data fed to an OPL emulator or converted to a playable format.
C#
Same approach via SDL3-CS audio API. OPL emulation library in C# or P/Invoke to native. TimeCount managed via Stopwatch or SDL ticks.
TypeScript
Web Audio API for sound output. OPL emulation in JavaScript/WASM. AudioWorklet for low-latency mixing. TimeCount from performance.now() scaled to 70Hz ticks.
Love2D
love.audio with SoundData for PCM output. OPL emulation in Lua or via FFI. TimeCount from love.timer.getTime() scaled to 70Hz. Music can use pre-rendered OGG/MP3 tracks.
