WL_DRAW.C — Raycasting Renderer Game

Purpose: The core 3D rendering engine. Implements raycasting to render walls, doors, and push-walls. Handles sprite projection and sorting, weapon overlay drawing, frame timing, and the triple-buffered page-flipping scheme. The raycaster uses DDA (Digital Differential Analyzer) to step through the tile grid for each screen column.

Performance critical: This is the most performance-sensitive file in the codebase. The original uses heavy inline assembly for the raycaster inner loop and column scaling. The SDL3 port replaces this with optimized C. For TypeScript/Lua ports, this is where the most optimization effort will be needed.

Porting Notes

C SDL3
The triple-buffer page system is replaced with a single 320x200 framebuffer. FixOfs becomes a no-op (or just calls VL_UpdateScreen). The raycaster core (DDA stepping) is pure math and translates directly. Wall/door column drawing writes to the linear framebuffer. The assembly AsmRefresh is replaced with WL_DR_A.C.
TypeScript
Raycasting math uses number (float64) or integer emulation with |0 truncation. Column drawing writes to Uint8Array or directly to ImageData. The main bottleneck is the per-column inner loop — consider WebAssembly for this if pure JS is too slow.
Love2D
Raycasting in Lua/LuaJIT. Column drawing via ImageData pixel operations. LuaJIT FFI can help with the inner loop. Alternative: use love.graphics draw calls for scaled columns if ImageData is too slow.
