ID_VL.C — Video Low-Level Engine

Purpose: Low-level VGA hardware interface. Handles video mode setup, palette management, pixel/line/bar drawing primitives, memory-to-screen blitting, latch operations, and screen fading. This is the lowest layer of the video system — everything between the game and the VGA hardware registers.

Author: John Carmack / id Software

Original target: DOS real-mode, VGA Mode-X (planar 320x200, 256 colors)

Side-by-Side: VL_Startup

C Original (DOS)
void VL_Startup (void)
{
    int i,videocard;

    asm cld;

    videocard = VL_VideoID ();
    for (i = 1;i < _argc;i++)
        if (US_CheckParm(_argv[i],ParmStrings) == 0)
        {
            videocard = 5;
            break;
        }

    if (videocard != 5)
        Quit ("Improper video card!  ...");
}
C SDL3 Port
void VL_Startup (void)
{
    if (sdl_window)
        return;

    if (!SDL_Init(SDL_INIT_VIDEO | SDL_INIT_AUDIO))
        Quit("SDL_Init(VIDEO|AUDIO) failed!");

    sdl_window = SDL_CreateWindow(
        "Wolfenstein 3-D", 320*3, 200*3, 0);
    if (!sdl_window)
        Quit("SDL_CreateWindow failed!");

    sdl_renderer = SDL_CreateRenderer(
        sdl_window, NULL);
    sdl_texture = SDL_CreateTexture(sdl_renderer,
        SDL_PIXELFORMAT_XRGB8888,
        SDL_TEXTUREACCESS_STREAMING, 320, 200);

    SDL_SetTextureScaleMode(sdl_texture,
        SDL_SCALEMODE_NEAREST);
    memset(sdl_screenbuf, 0, sizeof(sdl_screenbuf));
}

Side-by-Side: VL_FadeOut

C Original (DOS)
void VL_FadeOut (int start, int end,
    int red, int green, int blue, int steps)
{
    int i,j,orig,delta;
    byte far *origptr, far *newptr;

    VL_WaitVBL(1);
    VL_GetPalette (&palette1[0][0]);
    _fmemcpy (palette2,palette1,768);

    for (i=0;i<steps;i++)
    {
        origptr = &palette1[start][0];
        newptr = &palette2[start][0];
        for (j=start;j<=end;j++)
        {
            orig = *origptr++;
            delta = red-orig;
            *newptr++ = orig + delta * i / steps;
            // ... green, blue similarly
        }
        VL_WaitVBL(1);
        VL_SetPalette (&palette2[0][0]);
    }
    VL_FillPalette (red,green,blue);
    screenfaded = true;
}
C SDL3 Port
// Same algorithm, but palette is applied via
// sdl_palette[] and rendered through SDL_Texture.
// VL_SetPalette copies to sdl_palette[].
// VL_WaitVBL calls SDL_Delay(vbls * 14).
// VL_UpdateScreen converts indexed pixels to XRGB8888
// using sdl_palette lookup, then presents via renderer.
//
// The fade logic itself is unchanged - it interpolates
// between palette1 and palette2 over N steps,
// calling VL_WaitVBL and VL_SetPalette each frame.

Porting Notes

C SDL3
Replace VGA register I/O with a 320x200 byte framebuffer (sdl_screenbuf[]). All plane-aware drawing becomes simple linear byte writes. VL_UpdateScreen converts indexed colors to XRGB8888 via palette lookup and uploads to an SDL texture. Latch memory simulated with a separate latchmem[] array. VGA wait-for-VBL becomes SDL_Delay.
C#
Use byte[] framebuffer with SDL3-CS bindings. Palette as byte[768] or Color[] array. Drawing primitives become simple array indexing. Consider using Span<byte> for performance-critical blitting.
TypeScript
Use Uint8Array(64000) as the indexed framebuffer. Convert to ImageData RGBA for canvas display. Palette as Uint8Array(768). All VGA plane logic disappears entirely.
Love2D
Use love.image.newImageData(320,200) for the framebuffer. Palette stored as a Lua table of {r,g,b} triples. Convert indexed buffer to RGBA ImageData on each frame, then draw with love.graphics.draw.
